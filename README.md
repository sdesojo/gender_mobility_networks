
# Gender Mobility Networks

This repository provides code and analysis resources for the study:

**Women mobility networks enable more efficient travel**

It contains data-processing pipelines, statistical analysis scripts, and figure notebooks used to quantify and interpret gender differences in mobility behavior, mobility-network structure, tour organization, and travel efficiency.

The repository is intended as a scientific companion resource for reproducibility and methodological transparency. The arXiv link will be added when available.

## Scientific Scope

The workflow centers on user-month mobility traces and evaluates, among other quantities:

- Activity and repertoire metrics (for example, number of visits and number of unique locations).
- Mobility-network topology and centrality structure.
- Journey/tour sequence properties.
- Distance-based efficiency metrics balancing travel cost and spatial reward.
- Gender differences under direct comparison and matched-comparison settings.

## Repository Organization

- `src/compute_metrics/`: PySpark modules to compute mobility and network metrics from stop-level mobility data.
- `src/compute_seq_tours/`: PySpark modules to build sequences/tours and derive sequence-level efficiency descriptors.
- `src/nearest_neighbor_matching/`: nearest-neighbor matching utilities to compare users under controlled activity/repertoire conditions.
- `src/statistical_analysis/`: statistical helper functions (bootstrap summaries, distributional comparisons, quantile analyses).
- `scripts/`: executable analysis scripts corresponding to the main stages of the paper.
- `notebooks/`: figure-generation notebooks (`fig1` to `fig4`).
- `figures/`: exported figure outputs.

## End-to-End Workflow

### 1. Compute user-month mobility and network metrics

The script `scripts/01_compute_metrics.py`:

- Loads stop-level mobility data and demographics.
- Applies local-time conversion and temporal aggregation.
- Optionally detects home/work anchors.
- Computes mobility metrics and mobility-network metrics.
- Writes per-group outputs and optional pooled tables for downstream analyses.

### 2. Analyze gender differences in activity-repertoire statistics (Figure 1)

The script `scripts/02_analyze_gender_diff_Nk.py` supports analyses such as:

- Distributional histograms.
- Bootstrap estimates of central tendency and uncertainty.
- Gender gaps across activity/repertoire quantiles.
- Distributional distance/statistics and robust dispersion tests.

### 3. Matched comparison for network metrics (Figures 2 and 4a)

The script `scripts/03_compute_nwmet_matching_byNk.py`:

- Performs nearest-neighbor matching between male and female users.
- Controls for activity/repertoire dimensions (`visits`, `locations`).
- Computes true vs. shuffled-reference differences in network observables.

### 4. Post-processing of matched network results (Figure 2)

The script `scripts/04_analyze_gender_diff_nwmet.py` aggregates matched outcomes, evaluates significance relative to shuffled baselines, and prepares analysis products used in figure construction.

### 5. Compute movement sequences and tours

The script `scripts/05_compute_sequences_tours.py`:

- Builds ordered stop sequences at the selected temporal resolution.
- Cleans consecutive repeated stops.
- Extracts tours/journeys and keystone locations.
- Derives sequence-distance and reward/cost descriptors.

### 6. Analyze gender differences in tours (Figure 3)

The script `scripts/06_analyze_gender_diff_tours.py` performs tour-level analyses including length/bin distributions and gender-gap statistics across tour classes.

### 7. Analyze gender differences in travel efficiency (Figure 4)

The script `scripts/07_anlayze_gender_diff_toureff.py` evaluates efficiency contrasts by reward/cost regimes and activity quantiles.

## Installation

The project uses a `src/` layout and is configured with `pyproject.toml`.

From the repository root:

```bash
pip install -e .
```

Main declared dependencies:

- `pyspark`
- `pyyaml`
- `tqdm`

Depending on your execution path (analysis scripts and notebooks), additional scientific Python packages may be required (for example `pandas`, `numpy`, `scipy`, `scikit-learn`, `matplotlib`, `jupyter`).

## Data and Configuration

This repository does not bundle raw mobility data. To reproduce results, provide:

- Stop-level mobility records partitioned by user groups.
- User-level demographic attributes including gender and country information.
- Project-specific YAML configuration files (for Spark resources, aggregation resolution, paths, and enabled metrics).

Most scripts expose editable `input_path` / `output_path` variables or config-driven path blocks. Update these locations to match your local or cluster environment.

## Reproducibility Notes

- The intended execution model is staged: compute metrics -> run statistical analyses -> generate figures.
- Several analyses include bootstrap or shuffled-reference procedures; run time depends on sample size and iteration counts.
- PySpark settings (memory/workers/tmp directories) are read from configuration and should be tuned to your infrastructure.
